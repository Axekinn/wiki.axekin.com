## 🚀 Lossless Scaling 소개

**Lossless Scaling**은 게임 및 에뮬레이터에서 성능과 시각적 품질을 크게 향상시킬 수 있는 강력한 **프레임 생성** 및 **업스케일링** 도구입니다. GPU 통합 솔루션(DLSS, FSR)과 달리 Lossless Scaling은 시스템 수준에서 작동하며 **모든 GPU**와 호환됩니다.

### 주요 장점

> **✨ Lossless Scaling을 사용하는 이유?**
> 
> - **FPS 부스트:** 중간 프레임을 생성하여 프레임 속도를 두 배/세 배로 늘립니다.
> - **시각적 향상:** 더 높은 해상도를 위한 고품질 업스케일링
> - **범용 호환성:** NVIDIA, AMD 및 Intel과 함께 작동
> - **에뮬레이터 최적화:** 특히 Switch, PS2, PS3에 효과적
> - **낮은 지연 시간:** 입력 지연을 최소화하도록 최적화된 기술

### 이상적인 사용 사례

**Lossless Scaling이 뛰어난 분야:**
- 네이티브 30 FPS로 실행되는 에뮬레이션 게임(포켓몬, 젤다 등)
- 리소스 집약적인 에뮬레이터(RPCS3, Yuzu)
- 성능 향상이 필요한 보급형 PC
- 1080p → 1440p/4K 업스케일링
- 전반적으로 더 부드러운 게임 플레이

## ⚙️ 최적화된 Lossless Scaling 설정 (2025)

### 우선순위별 권장 프로필

최신 시각적 테스트 및 비교를 바탕으로 목표(품질 또는 성능)에 따라 우선순위를 정해야 할 구성은 다음과 같습니다.

#### 1. 최고 품질 (이미지: 최고 / 성능: 느림)

```
유형: LSG 3.1
플로우 스케일: 90
승수: 4
성능: OFF
```
최상의 이미지 품질을 원하고 GPU가 이를 처리할 수 있는 경우 이 프로필을 사용하십시오.

#### 2. 균형 잡힌 품질/성능 (이미지: 균형)

```
유형: LSG 3.1
플로우 스케일: 65
승수: 4
성능: OFF
```
이 프로필은 부드러움과 선명도 사이의 훌륭한 절충안을 제공하며 대부분의 사용자에게 권장됩니다.

#### 3. 최고 성능 (이미지: 나쁨 / 성능: 가장 빠름)

```
유형: LSG 3.1
플로우 스케일: 65
승수: 4
성능: ON
```
특히 보급형 설정이나 매우 까다로운 게임에서 FPS를 극대화하려면 이 설정을 우선적으로 사용하십시오.

> **팁:** 이전 버전(LSG 1.0)은 이제 구식이므로 피해야 합니다.

### 통합 팁

- 하드웨어 및 우선순위(품질 또는 성능)에 따라 프로필을 조정하십시오.
- 대부분의 사용자에게는 "균형" 프로필이 권장됩니다.
- 설정에 가장 적합한 절충안을 찾기 위해 다양한 설정을 테스트하는 것을 주저하지 마십시오.
- 이중 업스케일링을 방지하기 위해 에뮬레이터의 내부 스케일링과 Lossless Scaling을 동시에 사용하지 마십시오.

## 🔧 문제 해결

### 일반적인 문제 및 해결책

#### 과도한 지연 시간 (입력 지연)

**증상:** 작업과 화면 응답 사이에 눈에 띄는 지연이 있습니다.

**해결책 순서:**

```
1. 프레임 생성 승수 낮추기
   x4 → x3 → x2 → OFF
   
2. 프레임 생성 모드 전환
   LS1 → LSFG
   
3. 에뮬레이터에서 VSync 비활성화
   
4. 스케일링 팩터 낮추기
   2x → 1.5x → 1.2x
   
5. 캡처 방법 변경
   자동 → DXGI 데스크톱 복제 → Windows 그래픽 캡처
   
6. 프로세스 우선 순위 높이기
   보통 → 높음 → 실시간 (테스트 전용)
```

**"낮은 지연 시간" 구성:**
```yaml
스케일링 유형: AMD FSR
스케일링 팩터: 1.2x
프레임 생성: LSFG x2만
캡처 방법: DXGI 데스크톱 복제
우선 순위: 높음
에뮬레이터 VSync: OFF
```

#### 시각적 아티팩트 및 고스팅

**증상:** 잔상, 고스트 이미지, 번진 텍스처.

**해결책:**

**고스팅(잔상)의 경우:**
```
1. 프레임 생성 승수 낮추기
2. LS1에서 LSFG로 전환
3. 선명도 높이기 (0.8 → 1.0)
4. 게임이 안정적인 네이티브 FPS로 실행되는지 확인
5. 에뮬레이터에서 모션 블러 비활성화
```

**그래픽 아티팩트의 경우:**
```
1. 스케일링 유형 변경 (FSR → NIS 또는 그 반대)
2. 선명도 낮추기 (0.9 → 0.6)
3. 캡처 방법 변경
4. GPU 드라이버 업데이트
5. 오버레이 비활성화 (Discord, Steam 등)
```

**"품질 우선" 구성:**
```yaml
스케일링 유형: NIS
스케일링 팩터: 최대 1.5x
프레임 생성: LSFG x2
선명도: 0.7
캡처 방법: DXGI 데스크톱 복제
```

## ❓ FAQ - 자주 묻는 질문

### 일반적인 질문

**Q: Lossless Scaling은 모든 에뮬레이터에서 작동합니까?**  
A: 예, Lossless Scaling은 시스템 수준에서 작동하며 모든 Windows 에뮬레이터와 호환됩니다. 결과는 에뮬레이터와 게임에 따라 다를 수 있습니다.

**Q: Lossless Scaling을 사용하는 것이 합법적입니까?**  
A: 물론입니다. Steam에서 판매되는 공식 소프트웨어이며 모든 Windows 응용 프로그램의 디스플레이를 향상시킵니다.

**Q: 에뮬레이터의 네이티브 DLSS/FSR과 함께 사용할 수 있습니까?**  
A: 예, 하지만 일반적으로 권장되지 않습니다(이중 업스케일링). 최상의 결과를 얻으려면 둘 중 하나를 선택하십시오.

### 기술적인 질문

**Q: LSFG와 LS1의 차이점은 무엇입니까?**  
A: LSFG(Lossless Scaling Frame Generation)는 최신 기술로, 더 효율적이고 지연 시간이 짧습니다. LS1은 이전 세대이며 LSFG가 불안정한 경우에 사용합니다.

**Q: 내 AMD/Intel GPU에서 Lossless Scaling을 사용할 수 있습니까?**  
A: 예! DLSS(NVIDIA 전용)와 달리 Lossless Scaling은 모든 GPU(NVIDIA, AMD, Intel)에서 작동합니다.

**Q: 얼마나 많은 VRAM이 필요합니까?**  
A: 일반적으로 최소 4GB, 권장 6-8GB입니다. 업스케일링 및 프레임 생성에는 추가 VRAM이 사용됩니다.

**Q: 지연 시간이 정말 눈에 띄나요?**  
A: LSFG x2를 사용하면 추가되는 지연 시간이 최소화됩니다(5-10ms). 경쟁 게임에서는 눈에 띄지만 솔로/에뮬레이션에서는 무시할 수 있습니다.

**Q: 네이티브 PC 게임에서 Lossless Scaling을 사용할 수 있습니까?**  
A: 예, Lossless Scaling은 모든 Windows 게임 또는 응용 프로그램에서 작동합니다.
