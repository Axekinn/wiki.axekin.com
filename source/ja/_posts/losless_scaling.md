## 🚀 Lossless Scalingの紹介

**Lossless Scaling**は、ゲームやエミュレータのパフォーマンスと視覚品質を大幅に向上させることができる、**フレーム生成**と**アップスケーリング**のための強力なツールです。GPU統合ソリューション（DLSS、FSR）とは異なり、Lossless Scalingはシステムレベルで動作し、**すべてのGPU**と互換性があります。

### 主な利点

> **✨ なぜLossless Scalingを使用するのか？**
> 
> - **FPSブースト：** 中間フレームを生成してフレームレートを2倍/3倍にする
> - **視覚的向上：** 高解像度のための高品質アップスケーリング
> - **普遍的な互換性：** NVIDIA、AMD、Intelで動作
> - **エミュレータの最適化：** 特にSwitch、PS2、PS3で効果的
> - **低遅延：** 入力ラグを最小限に抑えるために最適化された技術

### 理想的な使用例

**Lossless Scalingが優れている場面：**
- ネイティブ30 FPSで実行されるエミュレートされたゲーム（ポケモン、ゼルダなど）
- リソースを大量に消費するエミュレータ（RPCS3、Yuzu）
- パフォーマンス向上を必要とする控えめなPC
- 1080p → 1440p/4Kへのアップスケーリング
- 全体的にスムーズなゲームプレイ

## ⚙️ 最適化されたLossless Scaling設定（2025年）

### 優先度別推奨プロファイル

最新の視覚テストと比較に基づき、目標（品質またはパフォーマンス）に応じて優先すべき設定は次のとおりです：

#### 1. 最高品質（画質：最高 / パフォーマンス：遅い）

```
タイプ：LSG 3.1
フロースケール：90
乗数：4
パフォーマンス：OFF
```
最高の画質を求め、GPUがそれを処理できる場合は、このプロファイルを使用してください。

#### 2. バランスの取れた品質/パフォーマンス（画質：バランス）

```
タイプ：LSG 3.1
フロースケール：65
乗数：4
パフォーマンス：OFF
```
このプロファイルは、滑らかさとシャープネスの優れた妥協点を提供し、ほとんどのユーザーに推奨されます。

#### 3. 最高パフォーマンス（画質：悪い / パフォーマンス：最速）

```
タイプ：LSG 3.1
フロースケール：65
乗数：4
パフォーマンス：ON
```
特に控えめな構成や非常に要求の厳しいゲームでFPSを最大化したい場合は、これを優先してください。

> **ヒント：** 古いバージョン（LSG 1.0）は現在廃止されており、避けるべきです。

### 統合のヒント

- ハードウェアと優先順位（品質またはパフォーマンス）に応じてプロファイルを調整してください。
- ほとんどのユーザーには、「バランス」プロファイルが推奨されます。
- セットアップに最適な妥協点を見つけるために、さまざまな設定を試すことをためらわないでください。
- 二重のアップスケーリングを防ぐために、エミュレータの内部スケーリングとLossless Scalingを同時に使用することは避けてください。

## 🔧 トラブルシューティング

### 一般的な問題と解決策

#### 過度の遅延（入力ラグ）

**症状：** アクションと画面上の応答の間に顕著な遅延がある。

**解決策（順に）：**

```
1. フレーム生成の乗数を下げる
   x4 → x3 → x2 → OFF
   
2. フレーム生成モードを切り替える
   LS1 → LSFG
   
3. エミュレータでVSyncを無効にする
   
4. スケーリング係数を下げる
   2x → 1.5x → 1.2x
   
5. キャプチャ方法を変更する
   自動 → DXGIデスクトップ複製 → Windowsグラフィックスキャプチャ
   
6. プロセスの優先度を上げる
   通常 → 高 → リアルタイム（テストのみ）
```

**「低遅延」設定：**
```yaml
スケーリングタイプ：AMD FSR
スケーリング係数：1.2x
フレーム生成：LSFG x2のみ
キャプチャ方法：DXGIデスクトップ複製
優先度：高
エミュレータのVSync：OFF
```

#### 視覚的アーティファクトとゴースティング

**症状：** 残像、ゴーストイメージ、ぼやけたテクスチャ。

**解決策：**

**ゴースティング（残像）の場合：**
```
1. フレーム生成の乗数を下げる
2. LS1からLSFGに切り替える
3. シャープネスを上げる（0.8 → 1.0）
4. ゲームが安定したネイティブFPSで実行されていることを確認する
5. エミュレータでモーションブラーを無効にする
```

**グラフィカルアーティファクトの場合：**
```
1. スケーリングタイプを変更する（FSR → NISまたはその逆）
2. シャープネスを下げる（0.9 → 0.6）
3. キャプチャ方法を変更する
4. GPUドライバを更新する
5. オーバーレイを無効にする（Discord、Steamなど）
```

**「品質第一」設定：**
```yaml
スケーリングタイプ：NIS
スケーリング係数：最大1.5x
フレーム生成：LSFG x2
シャープネス：0.7
キャプチャ方法：DXGIデスクトップ複製
```

## ❓ FAQ - よくある質問

### 一般的な質問

**Q: Lossless Scalingはすべてのエミュレータで動作しますか？**  
A: はい、Lossless Scalingはシステムレベルで動作し、すべてのWindowsエミュレータと互換性があります。結果はエミュレータやゲームによって異なる場合があります。

**Q: Lossless Scalingを使用することは合法ですか？**  
A: もちろんです。これはSteamで販売されている公式ソフトウェアで、任何のWindowsアプリケーションの表示を向上させます。

**Q: エミュレータのネイティブDLSS/FSRと併用できますか？**  
A: はい、しかし一般的には推奨されません（二重アップスケーリング）。最良の結果を得るには、どちらか一方を選択してください。

### 技術的な質問

**Q: LSFGとLS1の違いは何ですか？**  
A: LSFG（Lossless Scaling Frame Generation）は最新の技術で、より効率的で低遅延です。LS1は古い世代で、LSFGが不安定な場合に使用します。

**Q: 私のAMD/Intel GPUはLossless Scalingを使用できますか？**  
A: はい！DLSS（NVIDIAのみ）とは異なり、Lossless ScalingはすべてのGPU（NVIDIA、AMD、Intel）で動作します。

**Q: どのくらいのVRAMが必要ですか？**  
A: 一般的に最低4GB、推奨6〜8GBです。アップスケーリングとフレーム生成は追加のVRAMを使用します。

**Q: 遅延は本当に顕著ですか？**  
A: LSFG x2を使用すると、追加される遅延は最小限（5〜10ms）です。対戦ゲームでは顕著ですが、ソロ/エミュレーションでは無視できます。

**Q: ネイティブPCゲームでLossless Scalingを使用できますか？**  
A: はい、Lossless Scalingは任何のWindowsゲームまたはアプリケーションで動作します。
