## 🎮 melonDSの紹介

**melonDS**は、2016年から開発されている、現代的で正確なオープンソースのニンテンドーDSエミュレータです。優れた互換性、エミュレーションの精度、ローカルWi-Fi、セーブステート、グラフィックス向上などの高度な機能で際立っています。DeSmuMEに代わる完璧な現代的代替品です。

> 💡 **重要事項**：melonDSは、DSのシンプルなアーキテクチャのおかげで、ほとんどの現代的なエミュレータよりも要求が低いです。

## 📁 ステップ2：フォルダ構造

ファイルを整理するために、このフォルダ構造を作成します：

ライブラリを最適に整理するために、推奨されるツリー構造は次のとおりです：

````
DS Games/
├── おいでよ どうぶつの森/
│   ├── BASE/                # 🎮 ベースゲーム（メインファイル）
├── アノ1701：ディスカバリーの夜明け/
│   ├── BASE/                # 🎮 ベースゲーム
├── アートアカデミー/
│   ├── BASE/
└── アサシン クリード アルタイルクロニクルズ/
    ├── BASE/
````

### コンテンツの種類

- **BASE** 📁：ダウンロードするメインゲーム

✅ **自動検出**：ゲームは自動的にエミュレータに追加されます  
✅ **簡素化された管理**：各ROMを手動で選択する必要はありません  
✅ **明確な整理**：各ゲームのコンテンツを簡単に確認できます  
✅ **簡単なメンテナンス**：コンテンツの追加/削除が簡素化されます

> **📝 重要事項**：ゲームにアップデートやDLCがない場合、対応するフォルダを作成する必要はありません。実際に持っているコンテンツのフォルダのみを作成してください。

### 結果

この構造が整うと、エミュレータは「DS Games」フォルダ内のすべてのゲームを自動的に検出し、ライブラリに適切に整理します！ 🎯

## ⚙️ ステップ3：ニンテンドーDS BIOSのインストール

### 必要なBIOSファイル

> ⚠️ **（非）必須**：最適な機能のためには、ニンテンドーDSiのBIOSファイルが必要です。

**必要なファイル：**
- `biosdsi7.bin` (16 KB) - ARM7 BIOS
- `biosdsi9.bin` (4 KB) - ARM9 BIOS  
- `dsifirmware.bin` (256 KB) - DSファームウェア

**📥 ニンテンドーDSi BIOSのダウンロード：**

[ニンテンドーDSi BIOSをダウンロード](https://cdn.discordapp.com/attachments/1332956018665328702/1381293934109786246/Nintendo_DSi_Bios_for_emulators.zip?ex=6846fdb9&is=6845ac39&hm=e36162ccf2877cea45ea2ab9e004f2d38fddd8a7583be41f7e8414edc49e1383&)

> **📋 内容：** `biosdsi7.bin`, `biosdsi9.bin`, `dsifirmware.bin`

### melonDSでの設定

1.  `melonDS.exe`を**起動**
2.  `Config → Emu settings → DSi-mode`
3.  BIOSパスを**設定**：
    - **DS ARM9 BIOS**: `bios/biosdsi9.bin`
    - **DS ARM7 BIOS**: `bios/biosdsi7.bin`
    - **DS Firmware**: `bios/dsifirmware.bin`
4.  melonDSを**再起動**

## ⚙️ ステップ4：基本設定

### 一般設定
```
Config → Emu settings → General
```

**推奨設定：**
- **コンソールタイプ**：DS（DSiゲームの場合はDSi）
- **ダイレクトブート**：✅ 有効（ゲームを直接開始）
- **JIT有効化**：✅ 有効（パフォーマンスを大幅に向上）
- **JIT最大ブロックサイズ**：32（最適）
- **JITリテラル最適化**：✅ 有効
- **JIT分岐最適化**：✅ 有効
- **JIT高速メモリ**：✅ 有効

### 3D設定
```
Config → Video Settings
```

**表示設定**（左）とレンダリング設定**ソフトウェアレンダラ** / **OpenGLレンダラ**（右）の2つの主要セクションがあるウィンドウにアクセスします。

#### 3Dレンダリング設定

**レンダリングエンジンの選択：**

| オプション | パフォーマンス | 品質 | 推奨 |
|--------|-------------|---------|----------------|
| **ソフトウェア** | 低 | 基本 | 古いハードウェア |
| **OpenGL（クラシック）** | 優秀 | 非常に良い | ✅ **推奨** |
| **OpenGL（コンピュートシェーダー）** | 可変 | 最大 | 最近のGPUのみ |

#### OpenGL設定（推奨）

**内部解像度：** GPUに応じて`1xネイティブ（4096x3072）`から`16x`まで
- 1x = オリジナルのDS解像度
- 2x-3x = 品質の良い/パフォーマンスの妥協点  
- 4x+ = 強力なGPUが必要

**強化オプション：**
- ✅ **改善されたポリゴン分割**：ジオメトリの品質を向上
- ✅ **高解像度座標を使用**：テクスチャの精度を向上

#### 一般表示設定

```
表示設定（左側）
```

**推奨設定：**
- **VSync**：✅ 有効（ティアリングを防ぐ）
- **VSync間隔**：`1`（標準60 FPS）
- **OpenGLディスプレイ**：✅ OpenGL表示のためにチェック

### 視覚的な結果

これらの設定により、DSゲームは**解像度が向上**し、元のコンソールと比較して**表示品質が大幅に向上**します！

> **💡 パフォーマンスのヒント**：速度低下が発生した場合は、レンダリングエンジンを変更する前に、まず**内部解像度**を下げてください。

## 🔧 トラブルシューティング

### 起動の問題

#### melonDSが起動しない
```bash
システムチェック：
1. Visual C++ Redistributableがインストールされている
2. GPUがOpenGL 3.2+をサポートしている
3. グラフィックドライバが最新である
4. アンチウイルス：melonDSフォルダの例外設定
5. 必要に応じて管理者権限
```

#### ROMが起動しない
```bash
ROM診断：
1. フォーマット：有効な.ndsか確認
2. サイズ：ファイルが破損していない
3. BIOS：ファイルが存在し、有効である
4. リージョン：PAL/NTSC/JPN互換
5. 保護：一部のダンプにはパッチが必要
```

## ❓ FAQ - よくある質問

### 一般的な質問

**Q: melonDSはすべてのDSゲームをプレイできますか？**
**A:** 約97%のゲームが完璧に動作します。全体的に優れた互換性です。

**Q: BIOSファイルは必要ですか？**
**A:** はい、最適な機能と最大の互換性のために必要です。

**Q: melonDSはDSiゲームをサポートしていますか？**
**A:** 部分的なサポートです。一部のDSiゲームは動作しますが、まだ動作しないものもあります。

### 技術的な質問

**Q: OpenGLとソフトウェアレンダラ、どちらが良いですか？**
**A:** パフォーマンスと視覚的な向上のためにOpenGLを推奨します。互換性のためにソフトウェアを使用します。

**Q: どの解像度を選ぶべきですか？**
**A:** 品質の良い/パフォーマンスの妥協点として2x-3x。強力なGPUがあれば4x+。

**Q: JITは安全ですか？**
**A:** はい、良好なパフォーマンスのために必要です。セキュリティリスクはありません。

### 一般的な問題

**Q: 良いPCなのにラグが発生するのはなぜですか？**
**A:** JITが有効になっているか確認し、OpenGLを使用し、必要に応じてスケールファクターを下げてください。

**Q: セーブが機能しませんか？**
**A:** セーブフォルダの書き込み権限、正しいROMフォーマットを確認してください。

**Q: Wi-Fiが機能しませんか？**
**A:** Wi-Fiはサポートされているゲームに限定されており、ネットワーク設定が必要です。
