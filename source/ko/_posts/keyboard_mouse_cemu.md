> **⚠️ 중요 참고**: 가능은 하지만 Cemu에서 키보드-마우스 경험은 컨트롤러만큼 즐겁지 않을 것입니다. 이 솔루션은 호환되는 게임패드가 없는 플레이어에게 적합합니다.

## 소개

**Cemu** 에뮬레이터에서 **키보드와 마우스**로 **Wii U** 게임을 플레이하려면 컨트롤러를 시뮬레이션하기 위해 타사 도구를 사용해야 합니다. 이 가이드에서는 **Mouse2Joystick** 및 **ViGEmBus**를 사용하여 이 솔루션을 구성하는 방법을 설명합니다.

이 방법은 특히 다음과 같은 경우에 유용합니다:
- 호환되는 컨트롤러가 없는 플레이어
- 게임패드에 투자하지 않고 빠른 게임 테스트
- 키보드-마우스에 적합한 특정 게임 장르

### 일반적인 과정

1. **다운로드** ViGEmBus 및 Mouse2Joystick
2. **설치** ViGEmBus (가상 컨트롤러 드라이버)
3. 선호도에 따라 **구성** Mouse2Joystick
4. 컨트롤을 인식하도록 **설정** Cemu
5. **테스트** 및 설정 조정

## 전제 조건 및 다운로드

### 필수 소프트웨어

**필수 응용 프로그램:**
- **ViGEmBus**: 가상 컨트롤러용 드라이버
- **Mouse2Joystick**: 키보드/마우스 신호를 컨트롤러 신호로 변환
- **Cemu**: Wii U 에뮬레이터 (최신 안정 버전)

### 다운로드

#### ViGEmBus (가상 컨트롤러 드라이버)

**🔗 [ViGEmBus 다운로드](https://github.com/ViGEm/ViGEmBus/releases)**

**다운로드 지침:**
1. 릴리스 페이지에 **액세스**
2. 최신 `.msi` 파일 **다운로드**
3. **예시**: `ViGEmBus_Setup_1.17.333.msi`

**ViGEmBus 기능:**
- **무료** 및 오픈 소스
- Windows 10/11과 **호환**
- **안정적**이며 정기적으로 업데이트됨
- Xbox 컨트롤러 시뮬레이션에 **필수**

#### Mouse2Joystick (컨트롤 변환기)

**🔗 [Mouse2Joystick 다운로드](https://github.com/memethyl/Mouse2Joystick/releases)**

**다운로드 지침:**
1. 최신 `.zip` 파일 **다운로드**
2. **예시**: `Mouse2Joystick_v2.3.1.zip`
3. 전용 폴더에 **압축 해제** (예: `C:\Tools\Mouse2Joystick\`)

**Mouse2Joystick 기능:**
- **포터블**: 설치 필요 없음
- **사용자 정의 가능**: 구성 가능한 키 매핑
- **경량**: 최소한의 성능 영향
- **실시간**: 즉각적인 움직임 변환

## 1단계: ViGEmBus 설치

### 설치 과정

#### 설치 프로그램 실행

1. 다운로드한 `.msi` 파일 **찾기**
2. **더블 클릭**하여 설치 시작
3. 메시지가 표시되면 관리자 권한 **수락**

#### 설치 구성

**설치 단계:**
1. **환영** → "Next" 클릭
2. **라이선스 계약** → 동의하고 "Next" 클릭
3. **대상 폴더** → 기본값으로 두고 "Next" 클릭
4. **설치 준비 완료** → "Install" 클릭
5. **완료** → "Finish" 클릭

**설치된 구성 요소:**
- 가상 버스 드라이버
- 필수 시스템 라이브러리
- 장치 관리 인터페이스

### 설치 확인

#### 장치 관리자에서 확인

1. "내 PC"에서 **마우스 오른쪽 버튼 클릭** → **장치 관리자**
2. "시스템 장치" **확장**
3. "ViGEmBus Device" 존재 **확인**


## 2단계: Mouse2Joystick 구성

### 압축 해제 및 첫 사용

#### 소프트웨어 준비

1. 다운로드한 `.zip` 파일로 **이동**
2. **마우스 오른쪽 버튼 클릭** → **모두 압축 풀기**
3. 영구 폴더 **선택** (예: `C:\Tools\Mouse2Joystick\`)
4. `Mouse2Joystick.exe` **실행**

### 인터페이스 및 기본 구성

#### 첫 실행

실행되면 Mouse2Joystick은 간단한 인터페이스를 표시합니다:


#### 권장 구성

**제안된 기본 설정:**

**"Mouse Settings" 섹션:**
```
Mouse Sensitivity X: 3.0
Mouse Sensitivity Y: 3.0
Mouse Deadzone: 0.1
Invert Y-axis: 선호에 따라
```

**"Controller Settings" 섹션:**
```
Controller Type: Xbox 360 Controller
Vibration: 비활성화 (불필요)
```

### 키 매핑

#### 컨트롤 구성

**권장 표준 매핑:**

| Wii U 액션 | 키보드 키 | 기능 |
|--------------|--------------|----------|
| **왼쪽 스틱** | WASD | 이동 |
| **오른쪽 스틱** | 마우스 | 카메라 |
| **A 버튼** | 스페이스 | 주 액션 |
| **B 버튼** | 왼쪽 Ctrl | 보조 액션 |
| **X 버튼** | E | 상호작용 |
| **Y 버튼** | Q | 메뉴/인벤토리 |
| **L/R** | Shift/Tab | 범퍼 |
| **ZL/ZR** | 왼쪽/오른쪽 클릭 | 디지털 트리거 |

#### 고급 설정

**마우스 감도:**
- **FPS 게임**: 높은 감도 (4.0-5.0)
- **어드벤처 게임**: 중간 감도 (2.5-3.5)
- **플랫폼 게임**: 낮은 감도 (1.5-2.5)

**데드존:**
- **마우스**: 0.05-0.15 (미세 움직임 방지)
- **키보드**: 0.0 (즉각적인 반응)

## 3단계: Cemu 구성

### 컨트롤 설정 액세스

#### Cemu에서 탐색

1. Cemu **실행**
2. **메뉴** "Options" → **"Input Settings"**
3. **"-"** 버튼으로 기존의 모든 컨트롤러 **제거**


### 기존 컨트롤러 제거

#### 구성 정리

**제거 단계:**
1. 나열된 모든 컨트롤러 **식별**
2. 각 컨트롤러에 대해 **"-"** 버튼 **클릭**
3. 메시지가 표시되면 제거 **확인**
4. 목록이 비어 있는지 **확인**


### 새 컨트롤러 추가

#### XInput 구성 (Mouse2Joystick)

**Mouse2Joystick이 열려 있는 상태에서:**
1. **"+"** 버튼 **클릭**
2. API로 "XInput" **선택**
3. 나타나야 할 "Controller 1" **선택**
4. 선택 **확인**


#### 키보드 구성 (추가 키)

**매핑되지 않은 키의 경우:**
1. 다시 **"+"** **클릭**
2. API로 "Keyboard" **선택**
3. "Controller 1" **선택**
4. 특정 Wii U 키 **구성**


### 최종 키 구성

#### Cemu에서 매핑

**권장 구성:**

**XInput 컨트롤러 (Mouse2Joystick):**
```
왼쪽 스틱: 캐릭터 이동
오른쪽 스틱: 카메라 제어
A 버튼: 주 액션
B 버튼: 뒤로/취소
X 버튼: 보조 액션
Y 버튼: 메뉴
L/R 범퍼: 빠른 트리거
L/R 트리거: 아날로그 트리거
```

**키보드 컨트롤러 (특수 키):**
```
플러스: Enter (메인 메뉴)
마이너스: Escape (일시 중지)
홈: Tab (홈 메뉴)
스크린샷: F12
```

## 결론

**Mouse2Joystick** 및 **ViGEmBus** 덕분에 **Cemu**에서 키보드와 마우스로 **Wii U** 게임을 플레이하는 것이 가능합니다. 모든 게임에 최적의 경험은 아니지만, 이 솔루션을 통해 컨트롤러에 투자하지 않고도 훌륭한 Wii U 카탈로그를 발견할 수 있습니다.


### 최종 팁

**더 나은 경험을 위해:**
- 적합한 게임으로 시작 (젤다, RPG)
- 감도를 조정하는 데 시간을 투자하세요
- 구성을 저장하세요
- 결국 컨트롤러 구매를 고려하세요

## FAQ - 자주 묻는 질문

**Q: 이 방법이 모든 Wii U 게임에서 작동하나요?**
A: 아니요, 일부 게임(마리오 카트, 스매시 브라더스 등)은 적절한 경험을 위해 컨트롤러가 절대적으로 필요합니다.

**Q: 이 구성을 다른 에뮬레이터에 사용할 수 있나요?**
A: 예, Mouse2Joystick은 XInput을 지원하는 대부분의 에뮬레이터에서 작동합니다.

**Q: 지연 시간이 중요한가요?**
A: 지연 시간은 일반적으로 낮지만 하드웨어 구성에 따라 달라질 수 있습니다.

**Q: Mouse2Joystick을 영구적으로 열어 두어야 하나요?**
A: 예, 컨트롤러 시뮬레이션을 유지하려면 플레이하는 동안 Mouse2Joystick을 열어 두어야 합니다.

**Q: 이 방법이 Cemu 성능에 영향을 미치나요?**
A: Mouse2Joystick은 리소스 소모가 매우 적기 때문에 영향은 미미합니다.
